The Problem

Local Commerce is Broken

Small local businesses lose billions to big e-commerce platforms every year — not because they can't compete on quality, but because they lack the tools.

😤 For Customers


🗺️
Can't discover local shops — no way to find what's around them beyond Google Maps
💸
No price transparency — can't compare prices across local shops without visiting each one
📦
Ordering is cumbersome — phone calls, WhatsApp messages, no tracking
🎯
No personalization — same generic experience regardless of preferences
❓
No trust signals — hard to know if a local shop is reliable or good quality

😫 For Shop Owners


📱
No digital presence — expensive to build a website or app; stuck on Justdial
📊
Zero business intelligence — no analytics on who buys, when, what's trending
📸
Catalog creation is painful — taking photos, writing descriptions, pricing takes hours
🏷️
Pricing guesswork — no visibility into what competitors are charging nearby
📣
Customer retention is hard — no tools to reach customers with offers or updates

India has 63 million+ MSMEs. Only ~12% have any digital commerce presence. NearShop bridges this gap.

AI / ML Engine

13 AI Features Powered by GPT-4o

Every AI call is tracked for cost and latency. The system uses GPT-4o for vision/complex tasks and GPT-4o-mini for fast NLP — optimizing intelligence vs. cost.

📸

Snap-and-List Cataloging

Take a photo of any product. GPT-4o Vision extracts name, category, price range, attributes, tags, and confidence score. Shelf scan detects multiple products at once.

GPT-4o Vision
🔍

Conversational Search

Type "red Nike shoes under 2000" in plain English. GPT-4o-mini parses it into structured filters: keywords, category, price, color, brand, sort order.

NLP Parsing
🎯

Content-Based Recommendations

Builds interest profiles from views, wishlist, purchases. Matches product category/subcategory/tags to user preferences. Falls back to local popularity.

Collaborative ML
👥

Collaborative Filtering

"People near you also bought" — co-purchase matrix from order history. Diversity-aware selection to avoid clustering recommendations from the same shop.

Matrix Factorization
📈

Trending Detection

Hyperlocal trending within adjustable radius. Weighted event scoring in 24h window: view=1, wishlist=3, add_to_cart=5, purchase=10.

Time-Series Events
💰

AI Pricing Intelligence

Suggests competitive prices using local market comparables, demand signals (views/wishlist ratio), and elasticity notes. Real-time competitive analysis.

GPT-4o + Analytics
😊

Review Sentiment Analysis

Extracts positives, negatives, and improvement suggestions from customer reviews. 6-hour response cache for performance. Actionable for shop owners.

GPT-4o-mini NLP
🎪

RFM Customer Segmentation

Recency/Frequency/Monetary segmentation into Champions, Loyal, At Risk, Lost, etc. Absolute thresholds for stability. Shop owners can target each segment.

RFM Model
🔦

Demand Gap Analysis

Identifies unfulfilled local demand by aggregating low-result search queries. Tells shops "customers near you are searching for X but not finding it."

Search Log Mining
🛒

Personalized Deals

Ranks deals by: discount depth × shop rating × interest match × visit history × recency × scarcity signal. Each user sees a different deal order.

Multi-Factor Scoring
📊

Catalogue Suggestions

Market basket analysis on order history. Finds high-confidence category co-occurrence pairs. Tells shops which product categories to add next.

Association Rules
🧑‍💼

AI Business Advisor

GPT-4o chat with full shop context: products, orders, revenue trends, top items. Answers "how do I increase sales this week?" with data-backed advice.

GPT-4o Chat
🔬

AI Cost Tracker

Every OpenAI call is logged: tokens, cost (USD), response time, error status. Admin dashboard shows spend by feature. Prevents runaway AI costs.

Cost Telemetry

What Can Be Improved

Current Gaps & Quick Wins

🔴 Critical Fixes

  • • Schema mismatch: image_url vs images[] in Product model breaking unified search
  • • Visual search (CLIP embeddings) — stored but no query path implemented
  • • Mobile delivery carousel not yet wired to backend delivery zones API
  • • Order tracking real-time updates (WebSocket connected but not wired to mobile)

🟡 Performance & Scale

  • • No PostGIS — Haversine in Python doesn't scale beyond ~50k shops
  • • AI recommendations recalculated on every request — needs background job + cache
  • • No CDN for product images — all served from R2 origin
  • • Collaborative filter builds matrix in-memory — needs Redis precomputation

🟢 UX Improvements

  • • Map view for shops (ShopsMapPage exists but needs Mapbox/Google Maps integration)
  • • Dark/light theme toggle for web app
  • • Onboarding tutorial for first-time business owners
  • • Social sharing for deals and products (WhatsApp deeplinks)

💡 High-Impact New Features

🎥 Video Commerce

TikTok/Reels-style short product videos with in-video purchase buttons. Shop owners film, AI generates captions and tags.

🤝 Group Buying

Customers in a locality pool together for bulk discount — e.g., "10 neighbours want the same TV, get 15% off." Social + commerce.

🚴 Delivery Network

On-demand local delivery via independent gig riders. Shops get delivery without owning a fleet. Customer gets 30-min local delivery.

🏪 B2B Marketplace

Shops can order wholesale from suppliers directly through NearShop. Closes the loop from manufacturer → distributor → retailer → customer.

🌐 WhatsApp Integration

Shop owners receive orders via WhatsApp Business API. Customers who don't use the app can still order. Widens the funnel massively in India.

🏆 Neighbourhood Leaderboards

Gamified neighbourhood rankings for shops (most orders, best rating, fastest delivery). Drives organic competition and PR for shops.
